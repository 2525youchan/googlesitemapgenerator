Introduction

Google Sitemap Generator is software that you install on your web server. Once installed, Google Sitemap Generator automatically generates Sitemaps by analyzing the way that users access your content. Google Sitemap Generator can also submit Web Sitemaps automatically.

This document is an overview of Google Sitemap Generator. For more information about the Sitemaps protocol, see http://www.sitemaps.org/. For information about Google-specific functions and Webmaster Tools, see http://google.com/support/webmasters/.

About Google Sitemap Generator

Typical Sitemap generators create Sitemaps by crawling your website, much as a search engine does, so those tools may not improve coverage. Crawling may not be the most efficient mechanism to discover new content in databases or large websites, so it may also fail to improve freshness.

Google Sitemap Generator takes a different approach: it monitors your web server traffic and detects updates to your website in near real time. The main features of Google Sitemap Generator are:

  • Creation of Sitemaps based on web server traffic, web server logs and web server files.
  • Efficient generation of web, video, news, mobile and code search Sitemaps.
  • Easy configuration. Once you’ve done the initial configuration, no further administration is needed. But if you do want to tweak the settings, a web-based UI is available for remote configuration.
  • Easy filtering of URLs, based on URL patterns.
  • Automatic calculation of certain metadata, such as modified time and change frequency.
  • Automatic submission of Sitemaps for web search.

Google previously released sitemapgen, a Python-based tool, to Sourceforge. Google Sitemap Generator is a next-generation tool that relies on web server filtering rather than crawling, provides enhanced features, and supports more formats. If you’re new to Sitemap generators, we suggest starting with the new tool.

Installation and Deployment Issues

Check out our system requirements and determine how best to deploy Google Sitemap Generator in your environment! For full installation information, refer to the Installation document.

System requirements

Google Sitemap Generator runs on a variety of operating systems and web servers. See the installation document for the Windows and Linux prerequisites.

Based on our performance tests, Google Sitemap Generator has minimal performance impact on a web server.

Load-balanced web servers

For a site that is served by multiple, load-balanced web servers, you can choose from the following deployment options:

  • Install Google Sitemap Generator on one web server, set up a new domain name for this web server, and submit your Sitemaps to Google by using cross-domain submission. This is Google’s recommended solution. However, note that cross- domain submissions may not work for other search engines.
  • Install Google Sitemap Generator on one web server and make sure that the generated Sitemaps are copied to the other web servers. This is Google’s secondary recommendation.
  • Install Google Sitemap Generator on all web servers, and configure them using the same settings. All the web servers will then generate Sitemaps.

Multiple sites on a single web server

If multiple sites share the same web server, you can install Google Sitemap Generator once on the web server for all sites. Use the Google Sitemap Generator configuration UI to specify the sites for which you want to generate Sitemaps, and to specify settings on a per- site basis.

Configuration Options

Google Sitemap Generator has the following configuration options:

  • Excluding URLs from a Sitemap file. Use the configuration UI to automatically exclude URLs that match specified patterns.
  • Removing user session keys or other per-user sensitive information. You can configure Google Sitemap Generator to replace specified URL parameters with fixed values, or to remove them completely.

For more information about configuration, see the configuration document.

Security Concerns

We know that you don’t want Google Sitemap Generator to send any private data to Google! The only information that is shared with Google is the Sitemaps file. Furthermore, we're releasing the product through the open source community, so that you can download and examine it.

However, please do note that you are responsible for scrubbing a Sitemaps file of all user data before submitting it to Google. To accomplish this, you perform test generations of Sitemaps files and examine them for any URLs that should not be included.
