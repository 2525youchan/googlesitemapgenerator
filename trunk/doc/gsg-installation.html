Windows Installation

This section provides installation instructions for Windows platforms.

Prerequisites

These are the prerequisites for installation:

  • System and web server:
    • Microsoft Windows 2003 Server English/Chinese, 32bit/64bit with Internet Information Service (IIS) 6.0
    • Microsoft Windows 2008 Server 32bit/64bit with IIS 7.0
  • Free disk space: The number of unique URLs on your website determines the disk space needed. Calculate the necessary disk space based on the formula of 1G of disk space for 1 million URLs.
  • Web server access: Administrator access to the web server, through a console or remote desktop.
  • Authorization for submitting News Sitemaps (Optional) : If your site submits News Sitemaps, you must verify that you are the site owner and be authorized to submit Google News content. If you haven't yet performed these steps, see the Webmaster Help Center for instructions.

Installation Process

Installation of Google Sitemaps Generator restarts the World Wide Web Publishing service. Be sure to perform the installation at a time when the restart won't cause a problem.

Follow these steps to install Google Sitemaps Generator:

  1. From http://code.google.com/p/googlesitemapgenerator, download the latest version of sitemap_setup.msi.
  2. Run the file to launch the setup program.
  3. Read and accept the license agreement.
  4. Specify an installation folder on a disk that has the necessary disk space. A progress bar appears while Google Sitemap Generator installs its Windows service and IIS filter and then restarts the World Wide Web Publishing service. Several windows may open and close during this time.
  5. When the "Installation Complete" message appears, click Close to remove it.

When the installation is done, a web browser opens to the address http://localhost:8181. Now you're ready to configure Google Sitemaps Generator. View the configuration document for more information.

Uninstallation Steps

  1. From the Windows Control Panel's Add or Remove Programs window, find “Google Sitemap Generator” and click Remove.
  2. When you are prompted "Are you sure you want to remove Google Sitemap Generator...," click Yes. As Google Sitemap Generator uninstalls its Windows service and IIS filter, and restarts the World Wide Web Publishing service, you'll see a progress bar.
  3. When the "Installation Complete" message appears, click Close to remove the message. If you plan to reinstall Google Sitemap Generator in the future, you're done!
  4. If you don't plan to reinstall Google Sitemap Generator in the future, you can manually remove the historical data and the configuration file. These files are in website-specific directories that are under C:\Program Files\Google\Google Sitemap Generator unless you specified an alternative path during installation.

Linux Installation

This section provides installation instructions for Linux platforms.

Prerequisites

You can install Google Sitemaps Generator on Linux systems of the following types:

  • Red Hat Enterprise Linux 3 (32b/64b) with Apache 1.3
  • Red Hat Enterprise Linux 4 (32b/64b) with Apache 2.0
  • SUSE Linux Enterprise Server 10.0 (32b/64b) with Apache 2.2
  • Debian etch r0 with Apache 2.2.3
  • Fedora 7 with Apache 1.3
  • Mandriva 2007 with Apache 2.2
  • CentOS 4.6 with Apache 2.0.52
  • Ubuntu 6.10 (32b/64b) with Apache 2.2

You'll need 100MB to 1G free disk space. The number of unique URLs on your website determines the disk space needed. You can calculate the necessary disk space based on the formula of 1G of disk space for 1M URLs.

Installation Process

Follow these steps to install Google Sitemaps Generator:

  1. From http://code.google.com/p/googlesitemapgenerator, download the sitemap_install_i386.tar.gz or sitemap_install_x86_64.tar.gz
  2. Using the root account, extract the install package, as follows: 
    sudo tar ¨Czxvf  sitemap-install.tar.gz
  3. Run the install script. You'll need to know where the Apache binary is. You can enter the command with the path to the Apache binary or without it.
    • If you have the path to the Apache binary, you can enter it as follows: 
      sudo sitemap-install/install.sh ¨C-apache-bin=your_apache_binary_path
    • Alternatively, you can omit the apache-bin flag, and enter the command as follows: 
      sudo sitemap-install/install.sh
      The install script finds Apache, and asks you to confirm:

      Which is apache bin?[/usr/sbin/apache2]
      Press Enter to accept the specified value, or specify a different path. If there are multiple Apache servers installed, be careful that the displayed path is correct and enter the correct path if the displayed path is incorrect.

Results of Installation

This section describes the changes that the installation makes to your system.

Files installed

Installation of Google Sitemaps Generator creates the following files:

Files Created by Installing Google Sitemaps Generator
File Description
/usr/share/google-sitemap-generator Path to the program files.
/var/spool/google-sitemap-generator Path of internal database for URL storage. In the running process, this directory uses 100M to 1G disk space, according to the number of URLs.

/usr/sbin/google-sitemap-generato-ctl Control script of the program.

These files exist at runtime only:

  • /var/lock/google-sitemap-generator
  • /var/run/google-sitemap-generator.pid

Changes to the Apache configuration file

During installation, the installation script first backs up the Apache configuration file and then modifies the configuration file. The installation script does the following:

  • Backs up the configuration file to this location: /usr/share/google-sitemap-generator/httpd.install.conf
  • Adds the following line to enable Apache to load the Sitemap filter: 
    LoadModule google_sitemap_genrator_module  /usr/share/google-sitemap-generator/mod_sitemap.so

During uninstallation, the change is rolled back and the modified configuration file is saved here:
/usr/share/google-sitemap-generator/httpd.uninstall.conf.

The control script

Once installation is complete, Google Sitemaps Generator service will automatically start when the system starts. When the system is running, you can control the Generator service by running /usr/sbin.google-sitemap-generator-ctl. You must have root privileges in order to run the script.

Use the following command to control the Google Sitemaps Generator service: 

sudo google-sitemap-generator-ctl ¨Cflag

The flag variable can have one of the following values:

  • restart
  • start
  • stop

Note that this control script controls the Google Sitemap Generator service, but does not control the Sitemap filter in Apache. To control the filter, you'll need to use the Apache controller.

Uninstallation Steps

To uninstall Google Sitemap Generator, use the root account to run the uninstall script, as follows: 

sudo  /usr/share/google-sitemap-generator/uninstall.sh

Special Configuration for News Sitemaps

If your site submits News Sitemaps, you must now configure Webmaster Central to accept the generated News Sitemaps. To configure Google Webmaster Tools, go to http://www.google.com/webmasters/tools and add a Sitemap of the News type.
